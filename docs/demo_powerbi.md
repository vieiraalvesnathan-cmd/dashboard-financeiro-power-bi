# Demonstração segura do Power BI

## Recomendação

Para um portfólio público, use apenas dados fictícios. Nunca publique um relatório conectado a dados pessoais, financeiros ou corporativos.

## Screenshots

É a opção inicial recomendada porque não exige licença adicional e permite que qualquer visitante entenda rapidamente o resultado.

1. Troque a fonte do relatório pela base fictícia.
2. Revise todas as páginas, filtros e tooltips.
3. Salve as capturas em `assets`.
4. Referencie as imagens no README.

## Template `.pbit`

Um template pode demonstrar páginas, visuais, modelo, medidas e consultas sem carregar os dados de um PBIX comum. Antes de publicar, revise nomes, filtros, parâmetros e metadados.

## Publish to web

Use somente com base 100% fictícia. O conteúdo publicado pode ser acessado por qualquer pessoa e pode expor dados subjacentes.

## Ordem recomendada

1. screenshots com dados fictícios;
2. documentação técnica;
3. template `.pbit` revisado;
4. Publish to web somente após uma auditoria completa.
